## 2. 关键技术决策 (Architectural Decisions)

* **动态加载与弹性模拟降级**:
  物理 `serialport` 二进制库容易因为 VS Code ABI 不一致或在纯云端/无硬件环境开发下导致加载崩溃。
  * 决策：采取动态 `require('serialport')` 并捕获错误，一旦失败自动切换为虚拟硬件模拟器 (`VirtualSerialSimulator`)。这保证了插件能正常启动、UI 控件交互通畅、Agent 逻辑回路完整，极大增强了系统的稳定性与容错率。
* **Webpack 二进制依赖处理**:
  直接让 Webpack 打包 C++ `.node` 扩展将导致严重构建错误。
  * 决策：在 `webpack.config.js` 中使用 `externals: { 'serialport': 'commonjs serialport' }`，将物理依赖交由 Node.js 运行时宿主去解析。
* **全局单例与串口资源锁 (Singleton `SerialManager`)**:
  VS Code 内部可能通过 `ConnectionManager` 开启多个并行的 Agent Client 会话。
  * 决策：`SerialManager` 作为全局唯一的实例存在于 `extension.ts` 中，被往下层层注入到各会话的 `AcpClientImpl` 中，确保对同一物理串口资源的排他式安全操作，防止争抢冲突。
* **Webview 状态持久化 (Ring Buffer & CSP)**:
  侧边栏切换或折叠会导致 Webview 被卸载重建。
  * 决策：在 `SerialMonitorViewProvider` 中实现环形日志缓冲区；利用标准 `nonce` 和 CSP (Content-Security-Policy) 注入脚本，一旦 Webview 重绘，立刻将缓冲日志冲刷过去，保障“零丢失”的观测体验。
* **利用 ACP 协议扩展机制 (Standard Ext Slot)**:
  不修改 ACP SDK 原生代码。
  * 决策：通过实现 SDK 内置的 `extMethod` 和 `extNotification` 处理自定义串口请求，满足协议标准性的同时优雅实现客制化扩展。

## 5. v0.3.0 Hotfix 修复轮（连接 / 日志读取空白问题）

### 现象
真实硬件 + Windows + F5/vsix 场景下，点 Connect 看似已连接（按钮变 Disconnect、命令输入框可用）但 textarea 全程空白，本地连接 banner 与硬件回执均不可见。

### 根因（多因合一）
0. **打包丢了 hoisted 运行时依赖**（**真凶**，由用户 ESP32 实测暴露）：`.vscodeignore` 的 allowlist 只 un-ignore 了 `node_modules/serialport/**` 与 `node_modules/@serialport/**`，但 `@serialport/bindings-cpp` 的真正 native loader `node-gyp-build`（以及 `debug` / `ms`）通常被 npm hoist 到 `node_modules/` 根级。安装后的 `.vsix` 里有 prebuild 的 `.node` 文件但没有加载器，`require('serialport')` 抛 `Cannot find module 'node-gyp-build'`，整条链路退化到 `VirtualSerialSimulator` → 用户在端口下拉里只看得到 `VSIM1/2/3`。
1. **autoOpen 与 listener 注册竞态**：`new SerialPort({path,baudRate})` 默认 `autoOpen:true`，open 事件可能在 `.on('open', ...)` 注册前 fire。
2. **缺少 DTR/RTS 握手**：CH340 / CP2102 / ESP32 USB-CDC 等大量串口芯片在 DTR=低/未设置时停留在 reset 状态，永远不会喷启动日志。
3. **`listPorts` 与 `connect` 各自独立 `require('serialport')`**：`usingVirtual` 状态可能反复重写，UI 显示与实际链路割裂。
4. **`close`/`error` 事件不回推 UI**：被对端断开或 OS 抢占时 UI 仍显示已连接，"看似已连接"实为 silent close。
5. **webview ringBuffer 与 restoreHistory race**：webview 重建瞬间 `view=undefined` 时新增 `onData` 既未入 buffer 也未推送，`'ready'` 时整体覆盖丢增量。
6. **默认端口 `/dev/cu.debug-console`** 是 macOS 字面量，跨平台不友好。

### 修复
- `.vscodeignore` 增加 `!node_modules/node-gyp-build/**`、`!node_modules/debug/**`、`!node_modules/ms/**`，确保 vsix 含完整 native 加载链。验证：`vsce package` 后解包 `.vsix`，`extension/node_modules/node-gyp-build/` 必须存在。
- `SerialManager.connect()` 改为 `autoOpen:false` + 显式 `port.open(cb)`，**listener 先注册再 open**，彻底消除竞态。
- open 成功后自动 `port.set({dtr:true, rts:true})`（由 `embedderMonitor.assertControlSignals` 控制）。
- `require('serialport')` 在扩展激活时一次性懒加载并缓存为 `loadResult`，`listPorts` / `connect` 共用同一份结果，单写 `currentMode`。
- 引入 `onStatus` EventEmitter，`connecting / connected / disconnected / error` 全状态广播到 UI；`close` / `error` 事件强制回推 webview。
- `SerialMonitorViewProvider` 的 `onData` 始终先入 ringBuffer（不依赖 `this.view`），`'ready'` 时整体 `restoreHistory`，前端 `bootstrapped` 标志防覆盖增量。
- `package.json` `defaultPort` 默认空值（运行时取第一个端口），新增 `assertControlSignals / fallbackToVirtualOnConnectError / verboseLog` 三个配置开关。
- 新增 `OutputChannel('Embedder Monitor')` 与命令 `acp.serial.diag`，所有事件链路全程留痕。
- webview 顶部新增 `REAL` / `VIRTUAL` 模式徽章；端口下拉显示 `path — manufacturer / friendlyName`，虚拟端口加 🟡 前缀。
- `VirtualSerialSimulator` 增加 `set(...)` 兜底以兼容 DTR/RTS 调用，避免虚拟模式下握手报错。
- `acp:serial_disconnect` 改为 `await`（disconnect 现在是异步 close 回调）。

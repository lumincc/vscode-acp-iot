# vscode-acp-iot "EMBEDDER MONITOR" 串口集成开发执行计划 (v0.3.0)

本计划旨在为 `vscode-acp-iot` 插件新增一个专用的**侧边栏视图 (Webview View)** —— "EMBEDDER MONITOR"，并支持 AI Agent 通过 ACP (Agent Client Protocol) 协议对串口进行交互与监控。

---

## 一、 系统架构设计

系统采用经典的分层架构，解耦 UI 渲染、底层串口管理和 ACP 协议层：

```
+--------------------------------------------------------------+
|                         VS Code IDE                          |
+--------------------------------------------------------------+
                                |
  [手工操作]                    |                  [Agent 交互]
  Webview UI (Sidebar View)     |                  AI Agent (e.g. Claude Code)
        |                       |                        |
        v                       |                        v
+-------------------------------+------------------------------+
|             SerialMonitorViewProvider (UI 桥梁)              |
|                             以及                             |
|               AcpClientImpl (ACP 协议适配层)                  |
+-------------------------------+------------------------------+
                                |
                                v
+--------------------------------------------------------------+
|                   SerialManager (串口生命周期)                |
+--------------------------------------------------------------+
                                |
                                v
                    +-----------------------+
                    |  serialport (系统物理)|
                    |          或           |
                    |   Virtual Simulator   |
                    +-----------------------+
```

### 1. 各核心模块职责

* **`SerialManager`** (底层服务):
  - 维护单个活跃的串口连接 (`SerialPort` 实例)。
  - 提供 `listPorts()`、`connect(path, baud)`、`disconnect()` 和 `send(data, newline)` 的标准异步接口。
  - 使用 `vscode.EventEmitter` 广播统一的 `onData` 数据事件。
  - **弹性容错设计**: 当平台编译不支持 `serialport` 二进制包或在纯仿真开发环境中，自动降级到内置的 **Virtual Serial Simulator** (虚拟串口模拟器)，保证插件在任意环境下不崩溃、UI可演示、ACP 调用全链路可通。

* **`SerialMonitorViewProvider`** (视图控制):
  - 实现 `vscode.WebviewViewProvider` 接口。
  - 渲染 "EMBEDDER MONITOR" HTML，提供黑灰科技质感的暗色主题 UI。
  - 转发前端界面的配置操作（连接、发送、清空、换行符配置等）到 `SerialManager`。
  - 订阅 `SerialManager.onData`，实时把物理/模拟串口日志流推送到 Webview 的日志显示区。

* **`AcpClientImpl`** (ACP 协议适配):
  - 通过实现 SDK 定义的 `extMethod` 与 `extNotification` 来对接 AI Agent。
  - 响应 Agent 自定义请求：
    - `acp:serial_connect` (参数: `port`, `baud`): 打开指定端口。
    - `acp:serial_disconnect`: 关闭连接。
    - `acp:serial_write` (参数: `data`, `newline`): 向串口写入命令。
  - 订阅 `SerialManager.onData`，当收到串口物理反馈时，通过 `extNotification('acp:serial_data', { data })` 将数据流式反向实时推送给 Agent，建立双向控制闭环。

## 二、 ACP 自定义串口协议规范

为了实现 AI Agent 级别的高度可控性，我们设计了一套轻量、标准的 ACP 扩展协议方法：

### 1. 串口连接 (`acp:serial_connect`)
* **方向**: Agent -> Client (Request)
* **参数**:
  ```json
  {
    "port": "/dev/cu.debug-console",
    "baud": 115200
  }
  ```
* **响应**: `{ "success": true }` 或报错。

### 2. 串口断开 (`acp:serial_disconnect`)
* **方向**: Agent -> Client (Request)
* **参数**: `{}`
* **响应**: `{ "success": true }`

### 3. 串口写入 (`acp:serial_write`)
* **方向**: Agent -> Client (Request)
* **参数**:
  ```json
  {
    "data": "help\n",
    "newline": "\n"
  }
  ```
* **响应**: `{ "success": true }`

### 4. 串口日志实时接收 (`acp:serial_data`)
* **方向**: Client -> Agent (Notification)
* **参数**:
  ```json
  {
    "data": "System boot up...\n[CPU-0] OK\n"
  }
  ```

## 四、 关键风险与应对策略

1. **Native Serialport 二进制编译失败风险**:
   - *风险*: VS Code 的 Node.js 运行时 version 与本地系统全局编译的 `serialport` 可能产生 ABI 不一致，导致插件激活时报 `is not a valid Win32 application` 或 Electron 架构不匹配错误。
   - *策略*: `SerialManager` 内部采用动态懒加载设计 (`try { require('serialport') } catch ...`)。在加载失败时自动透明降级为**智能硬件仿真模拟器**，这样即使在没有连接真实物理串口的机器上，Agent 仍可通过模拟串口收发数据，不影响核心 ACP 协议流。

2. **多 Agent 串口占用冲突**:
   - *风险*: 若有多个 Agent 实例（例如同时连了 Claude 和 Copilot），它们同时操作同一串口会报错。
   - *策略*: `SerialManager` 实现独占锁，任何时刻仅允许单次连接。当新连接请求到来而前连接未释放时，给出人性化提示并优雅断开旧连接。

3. **Webview 上下文销毁导致日志丢失**:
   - *风险*: VS Code 在用户折叠或隐藏侧边栏视图时，默认会销毁 Webview。再次展开时之前的串口日志全部丢失。
   - *策略*: 在 `registerWebviewViewProvider` 时设置 `{ retainContextWhenHidden: true }`，并且在 `SerialMonitorViewProvider` 中额外维护一个轻量级的内存日志缓冲区，保证 Webview 重新解析时能够自动回显最近的几百行历史串口数据。
